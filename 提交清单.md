# GitHub 提交清单

当前目录还不是 Git 仓库。提交到 GitHub 前，可以先执行：

```powershell
git init
git add .
git status
git commit -m "Initial commit"
git branch -M main
git remote add origin <你的 GitHub 仓库地址>
git push -u origin main
```

执行 `git add .` 后，请用 `git status` 再确认一次只包含下面这些应提交文件。

## 应提交

### 项目代码

- `ChatSystem.sln`
- `NuGet.Config`
- `.config/dotnet-tools.json`
- `src/ChatSystem.Core/`
- `src/ChatSystem.Server/`
- `src/ChatSystem.Desktop/`

### 数据库结构和迁移

- `Data/main.sql`
- `src/ChatSystem.Server/Data/Migrations/`
- `src/ChatSystem.Server/Data/ApplicationDbContext.cs`
- `src/ChatSystem.Server/Data/DatabaseInitializer.cs`
- `src/ChatSystem.Server/Data/DatabasePath.cs`

说明：`Data/chat.db` 是本地运行数据库，不建议提交。项目启动时会自动创建并初始化数据库。

### 项目说明文档

- `README.md`
- `提交清单.md`
- `桌面端新增功能指南.md`

## 不提交

- `*.docx`

报告文件仅作为本地参考，不放进 GitHub 仓库。

### 构建产物

- `bin_verify/`

- `.vs/`
- `.idea/`

- `*.slnLaunch.user`

### 本地依赖缓存

## 提交前检查

```powershell
git status --short
```

如果看到报告、PDF、Word 文档、数据库文件、`bin/`、`obj/`、`.vs/` 或 `.nuget/packages/` 出现在待提交列表里，先不要提交，检查 `.gitignore` 或执行：

```powershell
git rm --cached <文件路径>
```
